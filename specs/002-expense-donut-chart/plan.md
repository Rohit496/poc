# Implementation Plan: Expense Category Donut Chart

**Branch**: `002-expense-donut-chart` | **Date**: 2026-04-29 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/002-expense-donut-chart/spec.md`

## Summary

Add a List/Chart toggle to the expense tracker dashboard's category breakdown section. In "Chart" view, a pure SVG donut chart (`DonutChartComponent`) renders per-category spending proportionally. The selected view is persisted to `localStorage` via the existing `StorageService`. No new runtime dependencies are introduced.

## Technical Context

**Language/Version**: TypeScript 5.x (strict mode), Angular 17+  
**Primary Dependencies**: Angular standalone components, RxJS, `jest-preset-angular`  
**Storage**: `localStorage` via existing `StorageService`  
**Testing**: Jest + `jest-preset-angular` (`npm test` in `apps/expense-tracker/`)  
**Target Platform**: Browser (evergreen; desktop + mobile)  
**Project Type**: Single-page web application (Angular SPA)  
**Performance Goals**: Chart renders within 100 ms of toggle activation (SC-001)  
**Constraints**: No new `npm` runtime dependencies; SVG only (no Canvas, no chart libraries)  
**Scale/Scope**: 1–7 category slices; up to hundreds of expense records

## Phase 0: Research

**Status**: ✅ Complete — see [research.md](research.md)

Key decisions:
1. **SVG `<path>` arcs** — outer + inner arc joined by line segments, computed by `buildSlices()`
2. **Pure utility** `buildSlices()` in `chart.utils.ts` — slice computation decoupled from presentation
3. **Tooltip** — inline `<div>` positioned absolutely over SVG, shown via `activeSlice` internal state
4. **Toggle persistence** — `StorageService.get/set('expense-tracker:category-view')` in `DashboardComponent.ngOnInit` / `onToggleView()`
5. **Colour palette** — `CATEGORY_COLORS` named constant in `categories.ts`

### Design decisions summary

| Decision | Choice | Rationale |
|---|---|---|
| Slice computation location | `DashboardComponent` (via `map` on `categoryTotals$`) | Keeps `DonutChartComponent` purely presentational; matches existing reactive pattern |
| Component boundary | `DonutChartComponent` receives `CategorySlice[]` input | Independently testable without `ExpenseService` |
| Toggle state owner | `DashboardComponent.categoryView` property | Dashboard controls layout; child components stateless (Principle VII) |
| Persistence mechanism | `StorageService` (existing) | No new infrastructure; consistent with `STORAGE_KEY` pattern |
| SVG geometry | Pre-computed path strings passed as `@Input` | Separation of math from rendering; pure function testable without Angular |
| Tooltip | Internal `activeSlice` state on `DonutChartComponent` | Scoped to chart; no parent involvement needed |
| Default view | `'list'` | Preserves existing dashboard behaviour on first load |
